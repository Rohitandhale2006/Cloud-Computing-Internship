# Task 4 - Cloud Security Implementation

## Objective

Implement cloud security controls using Identity and Access Management (IAM), secure cloud storage, and data encryption on Google Cloud Platform.

## Services Used

- Google Cloud IAM
- Google Cloud Storage
- Cloud Monitoring
- Data Encryption

## Security Components Implemented

### 1. Identity and Access Management (IAM)

IAM was configured to control access to cloud resources.

Actions Performed:
- Reviewed IAM roles and permissions
- Applied the principle of least privilege
- Restricted unauthorized access
- Verified user permissions

### 2. Secure Cloud Storage

Cloud Storage security settings were configured to protect stored data.

Actions Performed:
- Created a private storage bucket
- Disabled public access
- Configured bucket permissions
- Managed access using IAM policies

### 3. Data Encryption

Data encryption was enabled to protect data at rest.

Actions Performed:
- Verified Google-managed encryption
- Ensured encrypted storage of uploaded files
- Reviewed encryption settings

### 4. Monitoring and Alerts

Security monitoring was configured to detect unusual activities.

Actions Performed:
- Created monitoring dashboard
- Configured alert policies
- Added email notification channel

## Security Architecture

User → IAM Authentication → Cloud Storage → Encryption Layer → Secure Data Storage

Monitoring and Alerting continuously observe system activity and generate notifications when thresholds are exceeded.

## Deliverables

- IAM Configuration
- Secure Storage Configuration
- Data Encryption Verification
- Security Architecture Diagram
- Security Documentation

## Security Benefits

- Access Control
- Data Protection
- Secure Storage
- Monitoring and Alerting
- Compliance with Cloud Security Best Practices

## Outcome

Successfully implemented cloud security controls including IAM policies, secure storage, data encryption, and monitoring using Google Cloud Platform.
